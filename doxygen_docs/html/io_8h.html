Detailed Description

Digital IO module public API.

Author
github.com/Baksi675

This module provides a named abstraction over GPIO pins, supporting both digital output (write) and digital input (read) operations. It follows the same subsystem lifecycle pattern used across the project: init → start → run → stop → deinit.

Typical usage:

  • Initialize the subsystem using io_init_subsys
  • Populate an IO_CFG_ts with a GPIO configuration and a name
  • Register a handle using io_init_handle
  • Start the subsystem using io_start_subsys
  • Drive or sample the pin using io_write and io_read
